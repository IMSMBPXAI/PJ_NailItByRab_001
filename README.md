# 💅 Nail It by Rabbitz - Nail Salon Booking System

![Project Status](https://img.shields.io/badge/Status-In_Progress-orange)
![Course](https://img.shields.io/badge/Course-Software_Requirements-blue)

ระบบเว็บไซต์จองคิวร้านทำเล็บเจล **Nail It by Rabbitz (สาขาหน้ามหาวิทยาลัยพะเยา)** พัฒนาขึ้นเพื่อแก้ปัญหาการจัดการคิวแบบ Manual ลดความผิดพลาดจากการรับคิวซ้อน (Double Booking) และยกระดับประสบการณ์การจองของลูกค้าให้สะดวก รวดเร็ว แบบ End-to-End

โปรเจกต์นี้เป็นส่วนหนึ่งของรายวิชา **227141 ความต้องการซอฟต์แวร์ (Software Requirements)** คณะเทคโนโลยีสารสนเทศและการสื่อสาร มหาวิทยาลัยพะเยา

---

## 🎯 ปัญหาที่ระบบนี้เข้ามาแก้ไข (Pain Points Solved)
- **Double Booking:** ป้องกันคิวชนกัน 100% ด้วยระบบ Time-slot Locking อัตโนมัติ
- **Duration Miscalculation:** คำนวณเวลาทำเล็บรวมอัตโนมัติตามบริการที่เลือก (เช่น ถอดเล็บ + ทาสีเจล)
- **No-shows:** ลดปัญหาลูกค้าลืมนัดด้วยระบบ Notification แจ้งเตือน
- **Communication Gap:** ลูกค้าสามารถดูคิวว่าง (Real-time Availability) จองคิว และจ่ายมัดจำได้เองโดยไม่ต้องรอแอดมินตอบแชท

---

## ✨ ฟีเจอร์หลัก (Core Features)

### 👤 ฝั่งลูกค้า (Customer)
- **Real-time Availability:** ตรวจสอบตารางเวลาว่างของร้านและช่างได้ทันที
- **Smart Booking:** เลือกรับหลายบริการในครั้งเดียว โดยระบบจะคำนวณเวลาที่ต้องใช้ให้อัตโนมัติ
- **Deposit Payment:** ยืนยันคิวด้วยการชำระเงินมัดจำ (100 บาท) ผ่านระบบพร้อมอัปโหลดสลิป
- **Notification:** ระบบแจ้งเตือนเมื่อใกล้ถึงเวลานัดหมาย
- **Verified Review:** รีวิวและให้คะแนนช่างได้เฉพาะเมื่อสถานะการจองเป็น "เสร็จสิ้น" (Completed) เพื่อป้องกันรีวิวปลอม

### 💅 ฝั่งช่างทำเล็บและเจ้าของร้าน (Nail Artist / Salon Owner)
- **Schedule Management:** จัดการตารางงาน เปิด-ปิดรับคิวได้แบบยืดหยุ่น
- **Service & Price Management:** กำหนดหมวดหมู่บริการ ราคา และระยะเวลา (Duration) ของแต่ละบริการได้อย่างอิสระ
- **Portfolio Gallery:** อัปโหลดรูปภาพผลงานเพื่อประกอบการตัดสินใจของลูกค้า

## 📄 เอกสารวิเคราะห์และออกแบบระบบ (Documentation)
โปรเจกต์นี้ให้ความสำคัญกับการวิเคราะห์และออกแบบ Requirement อย่างเป็นระบบ โดยประกอบไปด้วย:
- **[SRS (Software Requirements Specification)]** - ข้อกำหนดความต้องการซอฟต์แวร์
- **[User Story & Use Case Description]** - การอธิบายการไหลของระบบและการโต้ตอบของผู้ใช้
- **[RTM (Requirements Traceability Matrix)]** - ตารางติดตามความต้องการเพื่อป้องกันช่องโหว่
- **[Test Case Documents]** - เอกสารทดสอบระบบแบบ EQP และ BVA (รองรับ Valid & Invalid Test)
